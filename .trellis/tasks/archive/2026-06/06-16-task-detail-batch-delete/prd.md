# 任务详情页：查看明细与批量删除

## Goal

在任务中心添加任务详情页功能，让用户能够查看单个任务的上传明细（文件列表、进度、字节数等），并支持批量删除历史任务记录，提升任务管理效率。

## What I already know

从用户描述：
* 需要查看上传明细：单个任务的文件列表、上传进度、字节数等详细信息
* 需要删除功能：单个删除 + 批量选择删除
* 批量删除需要全选功能

从代码库检查：
* 现有 TasksScreen.kt 有三个 Tab：活跃/已完成/失败
* 现有操作：暂停、恢复、取消、重试、单个删除
* 使用 Material 3 + Jetpack Compose
* TaskEntity 存储在 Room 数据库中
* 设计原则：信息优先、操作直达、批量高效

## Assumptions (temporary)

* 任务详情页应该从任务列表点击进入
* 文件列表信息需要从任务 payload 或新增字段获取
* 批量删除仅在"已完成"和"失败" Tab 中可用（活跃任务不应批量删除）

## Open Questions

### 1. 任务详情入口 ✅
**决定：点击整个任务卡片进入详情页**
* 操作面积大，误触率低
* 现有操作按钮（暂停/重试/删除）保持在卡片内，点击这些按钮不触发导航
* 点击卡片其他区域（标题、进度条、状态等）进入详情

### 2. 文件列表显示策略 ✅
**决定：展开显示所有子文件/子文件夹**
* 文件夹上传：通过 `DocumentFile.fromTreeUri()` 递归扫描，显示树形文件列表
* 单文件上传：显示单个文件的详细信息（名称、大小、状态）
* 处理边界情况：如果原文件已删除（上传后删除），显示"文件已删除，无法查看详情"提示

### 3. 批量删除 UI 模式 ✅
**决定：顶部工具栏添加"批量删除"按钮，点击进入选择模式**
* 默认状态：顶部工具栏显示"批量删除"按钮（仅在"已完成"和"失败" Tab）
* 选择模式：
  - 任务卡片左侧显示复选框
  - 顶部工具栏替换为：全选按钮、已选数量提示（如"已选 3 项"）、删除按钮、取消按钮
  - 点击"取消"退出选择模式，恢复默认状态

### 4. 批量操作范围 ✅
**决定：仅"已完成"和"失败" Tab 支持批量删除**
* "活跃" Tab（waiting/running/paused）：不显示"批量删除"按钮，避免误操作中断正在进行的任务
* "已完成" Tab：支持批量删除，快速清理历史记录
* "失败" Tab：支持批量删除，用户可以批量清理失败任务或保留查看错误信息

## Requirements

### 任务详情页
* 从任务列表点击卡片（非操作按钮区域）进入详情页
* 显示任务基本信息：
  - 任务标题、ID
  - 状态（用状态徽标显示）
  - 创建时间、更新时间
  - NAS 配置名称（如果有）
* 显示上传进度：
  - 进度条（活跃任务）
  - 已上传字节数 / 总字节数
  - 百分比
* 显示文件列表：
  - 文件夹上传：递归扫描 `DocumentFile`，显示树形文件列表（文件名、大小）
  - 单文件上传：显示文件详情（文件名、大小）
  - 如果原文件已删除：显示"文件已删除，无法查看详情"提示
* 显示错误信息（如果 `status == "failed"` 或 `errorMessage != null`）
* 提供单个任务操作按钮：
  - 删除（所有状态）
  - 重试（仅失败任务）

### 批量删除功能
* 仅在"已完成"和"失败" Tab 显示"批量删除"入口
* 点击"批量删除"进入选择模式：
  - 任务卡片左侧显示复选框
  - 顶部工具栏替换为：全选按钮、已选数量提示、删除按钮、取消按钮
* 全选功能：一键选中当前 Tab 所有任务
* 删除操作：
  - 点击"删除"按钮弹出确认对话框，显示删除数量
  - 确认后批量删除选中任务
  - 删除后刷新列表，退出选择模式
* 取消功能：退出选择模式，清除选择状态

## Technical Approach

### 任务详情页实现
1. 创建 `TaskDetailScreen.kt` 和 `TaskDetailViewModel.kt`
2. 在 `NavGraph.kt` 添加详情页路由，参数为 `taskId`
3. `TaskDetailViewModel` 提供：
   - 任务信息状态流（从 `TaskRepository` 获取）
   - 文件列表扫描逻辑（通过 `DocumentFile` 递归扫描）
   - 删除/重试操作方法
4. UI 布局：
   - 顶部 AppBar（标题 + 返回按钮）
   - 基本信息卡片（状态、时间、配置）
   - 进度卡片（进度条、字节数）
   - 文件列表卡片（可折叠的树形列表）
   - 错误信息卡片（条件显示）
   - 底部操作按钮区

### 批量删除实现
1. 在 `TasksViewModel` 添加：
   - `isSelectionMode: StateFlow<Boolean>` — 选择模式状态
   - `selectedTaskIds: StateFlow<Set<String>>` — 已选任务 ID 集合
   - `enterSelectionMode()` / `exitSelectionMode()` — 进入/退出选择模式
   - `toggleTaskSelection(taskId)` — 切换任务选择状态
   - `selectAll()` / `clearSelection()` — 全选/清除
   - `batchDelete(taskIds)` — 批量删除
2. 在 `TasksScreen.kt` UI 更新：
   - 顶部工具栏根据 `isSelectionMode` 切换显示
   - 默认模式：显示"批量删除"按钮（仅"已完成"/"失败" Tab）
   - 选择模式：显示全选、已选数量、删除、取消按钮
   - 任务卡片左侧条件显示复选框
3. 点击卡片逻辑：
   - 选择模式：切换选择状态
   - 默认模式：导航到详情页

### 数据库操作
* `TaskDao.kt` 已有 `delete(task)` 方法，批量删除调用多次或添加 `deleteByIds(ids: List<String>)`
